## Типы адресов в стеке TCP/IP
Для идентификации узлов в сети используются три типа адресов, работающих на разных уровнях модели OSI.

### Локальные адреса (MAC)
**Канальный (L2)**
Физический адрес сетевого адаптера. 48 бит (6 байт). 
Пример: 00:1A:2B:3C:4D:5E. 
Уникален в пределах локальной сети.
### Сетевые адреса (IP)
**Сетевой (L3)**
Логический адрес узла в сети.
IPv4: 32 бита. IPv6: 128 бит. 
Пример: 192.168.1.100. 
Глобально маршрутизируется.
### Доменные имена (DNS)
**Прикладной (L7)**
Символьное имя, понятное человеку.
Пример: www.example.com. 
Преобразуется в IP через DNS.

## Локальные адреса (MAC-адреса)
**MAC (Media Access Control)** — уникальный идентификатор сетевого адаптера, прошитый производителем.
### Формат MAC-адреса
48 бит (6 байт)
Запись в шестнадцатеричном формате
Разделители: двоеточие или дефис

Пример: 00:1A:2B:3C:4D:5E или 00-1A-2B-3C-4D-5E

### Структура адреса
**Первые 3 байта (24 бита):**
OUI (Organizationally Unique Identifier) — код производителя

**Последние 3 байта:**
Серийный номер устройства

**Пример:** 00:1A:2B = Intel Corporation
## Сетевые IP-адреса
**IP-адрес** — логический адрес узла в сети, обеспечивающий маршрутизацию на сетевом уровне.
### IPv4

- **Формат:** 32 бита (4 байта)
    
- **Запись:** десятичные октеты через точку
    
- **Пример:** 192.168.1.100
    
- **Диапазон:** 0.0.0.0 — 255.255.255.255
    
- **Всего адресов:** ~4.3 млрд

### IPv6

- **Формат:** 128 бит (16 байт)
    
- **Запись:** 8 групп по 4 hex-цифры через :
    
- **Пример:** 2001:0db8:85a3:0000:0000:8a2e:0370:7334
    
- **Сокращённо:** 2001:db8:85a3::8a2e:370:7334
    
- **Всего адресов:** 340 ундециллионов
## Классы IP-адресов (IPv4)

**Классовая адресация** — исторический способ разделения адресного пространства на классы A, B, C, D, E. Сейчас заменён на CIDR (бесклассовую адресацию), но классы всё ещё используются для понимания.

| Класс | Диапазон                    | Маска по умолчанию  | Сети/Узлы                | Применение                     |
| ----- | --------------------------- | ------------------- | ------------------------ | ------------------------------ |
| A     | 1.0.0.-126.255.255.255      | 255.0.0.0 (/8)      | 128 сетей / 16 млн узлов | Крупные организации            |
| B     | 128.0.0.0-191.255.255.255   | 255.255.0.0 (/16)   | 16K сетей / 65K узлов    | Средние организации            |
| C     | 192.0.0.0 — 223.255.255.255 | 255.255.255.0 (/24) | 2 млн сетей / 254 узла   | Малые сети                     |
| D     | 224.0.0.0 — 239.255.255.255 | -                   | -                        | Multicast (групповая рассылка) |
| E     | 240.0.0.0 — 255.255.255.255 | -                   | -                        |                                |
## Подсети и маски подсетей

**Маска подсети** — определяет, какая часть IP-адреса относится к сети, а какая — к узлу. Позволяет разбивать большую сеть на подсети (subnetting).

### Формат маски

- **Десятичная запись:** 255.255.255.0
    
- **Префиксная запись (CIDR):** /24 (24 бита сети)
    
- **Двоичная запись:** 11111111.11111111.11111111.00000000

### Пример разбиения

- **IP:** 192.168.1.100
    
- **Маска:** 255.255.255.0 (/24)
    
- **Адрес сети:** 192.168.1.0
    
- **Адрес узла:** 100
    
- **Broadcast:** 192.168.1.255
    
- **Доступно узлов:** 254 (1–254)

**Формула:** $Количество узлов в подсети = 2∧(32−длина префикса)−22∧(32−длина префикса)−2. Вычитаем 2 (адрес сети + broadcast).$
## Назначение IP-адресов

### Автономная сеть (статическая)

Администратор вручную назначает IP-адрес каждому устройству.

**Преимущества:**

- Полный контроль
    
- Нет зависимости от DHCP-сервера

**Недостатки:**

- Трудоёмко для больших сетей
### DHCP (динамическая)

Dynamic Host Configuration Protocol — автоматическое назначение IP через сервер.

**Преимущества:**

- Автоматизация
    
- Минимум ошибок

**Недостатки:**

- Зависимость от сервера

## Отображение IP → MAC (ARP)

**ARP (Address Resolution Protocol)** — протокол разрешения адресов. Преобразует IP-адрес в MAC-адрес для доставки кадра в локальной сети.

### Процесс работы ARP

1. Узел A хочет отправить данные узлу B (знает IP, не знает MAC)
    
2. А отправляет ARP-запрос (broadcast): «Кто имеет IP 192.168.1.100?»
    
3. Узел B отвечает (unicast): «Это я, мой MAC: 00:1A:2B:3C:4D:5E»
    
4. А сохраняет соответствие IP ↔ MAC в ARP-кэше
    
5. А отправляет кадр с MAC-адресом B

### ARP-кэш и команды

**ARP-кэш** хранит соответствия IP ↔ MAC (~2 мин). Команды:
- Windows: arp -a (просмотр), arp -d (очистка)
- Linux/macOS: arp -n, ip neigh, arp -d
## Система DNS

**DNS (Domain Name System)** — распределённая система преобразования доменных имён в IP-адреса и обратно.

### Структура доменного имени

**Пример:** [www.example.com](https://www.example.com/)

- **www** — имя хоста (subdomain)
    
- **example** — домен второго уровня
    
- **com** — домен верхнего уровня (TLD)

Корневой домен (root): . (неявно)

### Процесс разрешения DNS

1. Клиент запрашивает у локального DNS-сервера
    
2. Если в кэше — возвращает
    
3. Иначе запрашивает у корневого сервера
    
4. Корневой перенаправляет к TLD-серверу (.com)
    
5. TLD перенаправляет к авторитативному серверу ([example.com](https://example.com/))
    
6. Авторитативный возвращает IP-адрес
    
7. Результат кэшируется (TTL)

**Типы записей:** A (IPv4), AAAA (IPv6), CNAME (псевдоним), MX (почта), NS (name server), TXT (текст)
## Доменные имена

**Доменное имя** — символьное обозначение узла в сети, удобное для человека. Используется вместо IP-адреса.
### **Популярные TLD**

- .com (коммерческие)
- .org (организации)
- .net (сетевые)
- .edu (образование)
- .gov (правительство)
- .ru, .uk, .de (национальные)
