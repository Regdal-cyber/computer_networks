## Что такое протокол и стек протоколов?
**Протокол** — набор правил и соглашений для обмена данными между устройствами в сети. Определяет формат сообщений, порядок их передачи, обработку ошибок.

**Стек протоколов** — иерархическая структура протоколов, где каждый уровень выполняет определённые функции и взаимодействует только со смежными уровнями.

### Зачем нужны стеки?
- Модульность — можно заменить протокол одного уровня без изменения других
- Стандартизация — единые правила для взаимодействия разных производителей
- Упрощение разработки — разработчик работает только с API нужного уровня

## Стек OSI — эталонная модель
**OSI (Open Systems Interconnection)** — 7-уровневая модель, разработанная ISO в 1984 году. Носит эталонный (теоретический) характер.

| 7. Прикладной    |
| ---------------- |
| 6. Представления |
| 5. Сеансовый     |
| 4. Транспортный  |
| 3. Сетевой       |
| 2. Канальный     |
| 1. Физический    |
**OSI** служит основой для понимания сетевых архитектур, но на практике чаще используется модель TCP/IP.

## Другие стеки протоколов
### IPX/SPX
Стек компании Novell для сетей NetWare (1980-е – 2000-е).

I**PX (Internetwork Packet Exchange)** — сетевой протокол (аналог IP)
**SPX (Sequenced Packet Exchange)** — транспортный протокол (аналог TCP)

### NetBIOS / SMB
Протоколы Microsoft для локальных сетей Windows.

**NetBIOS** — API для сетевого взаимодействия (имена, сессии)

**SMB/CIFS** (Server Message Block) — протокол общего доступа к файлам и принтерам

### Современное состояние
**IPX/SPX** — практически не используется (вытеснен TCP/IP)

**NetBIOS/SMB** — до сих пор применяется в Windows-сетях, но работает поверх TCP/IP (NetBIOS over TCP/IP). Современная версия — SMB 3.x с шифрованием.

## Стек протоколов TCP/IP
**TCP/IP** — практический стек протоколов Интернета. Разработан в 1970-х в рамках проекта ARPANET. Содержит 4 уровня.

| Прикладной        | HTTP, FTP, SMTP, DNS, SSH, Telnet | OSI: 5, 6, 7 |
| ----------------- | --------------------------------- | ------------ |
| Транспортный      | TCP, UDP                          | OSI: 4       |
| Cетевой(Internet) | IP, ICMP, ARP, IGMP               | OSI: 3       |
| Канальный (Link)  | Enternet, Wi-Fi, PPP              | OSI: 1, 2    |
## Назначение протоколов TCP/IP
### IP (Internet Protocol)
Маршрутизация пакетов, адресация (IPv4/IPv6)
### ICMP (Internet Control Message Protocol)
Диагностика сети (ping, traceroute)
### ARP (Address Resolution Protocol)
Разрешение IP → MAC адрес
### TCP (Transmission Control Protocol)
Надёжная доставка данных с подтверждением
### UDP (User Datagram Protocol)
Быстрая доставка без гарантий (DNS, видео)

## HTTP / HTTPS — протокол веба
**HTTP (Hypertext Transfer Protocol)** — основа World Wide Web. Используется для запроса и передачи веб-страниц, изображений, API.
### Основные методы HTTP
GET — запрос данных с сервера
POST — отправка данных на сервер
PUT — обновление ресурса
DELETE — удаление ресурса
### HTTPS = HTTP + SSL/TLS
HTTPS — защищённая версия HTTP с шифрованием трафика. Обязателен для:
- Аутентификация (логины/пароли)
- Платёжные данные
- Конфиденциальная информация
## FTP, Telnet, SSH

### FTP (File Transfer Protocol)
Протокол передачи файлов между клиентом и сервером. Поддерживает аутентификацию, управление каталогами, докачку файлов.

**Недостаток:** передача паролей в открытом виде. Современная альтернатива — SFTP (SSH File Transfer Protocol) или FTPS (FTP over SSL).
### Telnet (порт 23)
Протокол удалённого доступа к терминалу. Позволяет управлять удалённым сервером через командную строку.
### SSH (Secure Shell, порт 22)
Защищённый протокол удалённого доступа. Весь трафик шифруется. Поддерживает туннелирование (проброс портов), передачу файлов (SFTP).

SSH полностью заменил Telnet в современных системах.

## Протоколы электронной почты
Электронная почта использует несколько протоколов: для отправки (SMTP) и для получения (POP3 / IMAP).
### SMTP (Simple Mail Transfer Protocol, порт 25/587)
Протокол отправки писем с клиента на почтовый сервер и между серверами. Работает по схеме: клиент → сервер отправителя → сервер получателя.
### POP3 (Post Office Protocol, порт 110)
Протокол получения почты. Скачивает письма на клиент и удаляет их с сервера. Не синхронизирует между устройствами. Подходит для одного устройства.
### IMAP (Internet Message Access Protocol, порт 143)
Протокол получения почты с хранением на сервере. Письма синхронизируются между всеми устройствами. Поддерживает папки, флаги, поиск. Современный стандарт.
