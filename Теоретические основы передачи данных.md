## Понятие сигнала и данных
**Данные** - информация, представленная в дискретном (цифровом) виде, пригодном для обработки, хранения и передачи.

**Примеры:**
Текст, числа, изображения
Биты: 0 и 1 (двоичная система)

**Сигнал** - физический процесс (электрический, оптический, радиоволновый), переносящий данные по каналу связи.

**Типы:**
Аналоговый — непрерывный
Цифровой — дискретный

## Методы кодирования данных
**Кодирование** — преобразование данных (битов) в последовательность сигналов для передачи по физической среде.

### NRZ (Non-Return to Zero)
Простейшее кодирование: 1 = высокий уровень, 0 = низкий. Проблема: нет синхронизации при длинных последовательностях одинаковых битов.

### Manchester
Переход в середине такта: ↓ = 0, ↑ = 1. Самосинхронизация, но требует удвоенной полосы. Применяется в Ethernet 10 Мбит/с.

### B/5B
Каждые 4 бита кодируются 5 битами для гарантии частых переходов. Используется в Fast Ethernet (100 Мбит/с).

### 8B/10B
8 бит → 10 бит. Обеспечивает баланс DC и частые переходы. Применяется в Gigabit Ethernet, Fiber Channel.

## Модуляция сигналов
**Модуляция** — изменение параметров несущей волны (амплитуды, частоты, фазы) в соответствии с передаваемыми данными. Используется для передачи цифровых данных по аналоговым каналам (телефонные линии, радио).

### Основные виды модуляции
- **ASK (Amplitude Shift Keying)** — Амплитудная модуляция — изменяется амплитуда несущей. Чувствительна к помехам.
- **FSK (Frequency Shift Keying)** — Частотная модуляция — изменяется частота несущей. Более устойчива к шумам. Применяется в модемах.
- **PSK (Phase Shift Keying)** — Фазовая модуляция — изменяется фаза несущей. Высокая помехоустойчивость. Используется в Wi-Fi, LTE.
- **QAM (Quadrature Amplitude Modulation)** — Комбинация амплитудной и фазовой модуляции. Высокая скорость передачи. Применяется в кабельных модемах, DSL, LTE.

## Методы оцифровки (дискретизации)
**Оцифровка** — преобразование аналогового сигнала в цифровой. Применяется для передачи голоса, музыки, видео по цифровым каналам.
### Этапы оцифровки
1. **Дискретизация (Sampling)** 
   Измерение значения сигнала через равные промежутки времени (частота дискретизации).
2. **Квантование (Quantization)** 
   Округление измеренных значений до ближайшего уровня из заданного набора.
3. **Кодирование (Encoding)** 
   Представление квантованных значений в виде двоичных кодов.

### Примеры стандартов
- **PCM (Pulse Code Modulation)** 
  Базовый метод оцифровки. Применяется в телефонии (8 кГц, 8 бит → 64 кбит/с).
- **Audio CD** 
  44.1 кГц, 16 бит, стерео → 1411 кбит/с
- **MP3, AAC** 
  Сжатие с потерями для экономии полосы

## Понятие коммутации
**Коммутация** — процесс установления соединения между отправителем и получателем через промежуточные узлы сети. Определяет, как данные проходят путь от источника к назначению.

### Зачем нужна коммутация?
- **Масштабируемость** — невозможно соединить каждое устройство с каждым напрямую (N² связей)
- **Эффективность** — совместное использование каналов связи несколькими абонентами
- **Гибкость** — возможность динамического выбора маршрута в зависимости от нагрузки или отказов

## Типы коммутации

|              | Коммутация каналов                                                           | Коммутация пакетов                                                           | Коммутация сообщений                                             |
| ------------ | ---------------------------------------------------------------------------- | ---------------------------------------------------------------------------- | ---------------------------------------------------------------- |
| Принцип      | Перед передачей данных устанавливается выделенный канал на всё время сеанса. | Данные разбиваются на пакеты. Каждый пакет передаётся независимо через сеть. | Сообщение целиком передаётся от узла к узлу (store-and-forward). |
| Преимущества | Постоянная полоса,<br>малая задержка                                         | Эффективное использование каналов,<br>отказоустойчивость                     | Не требует постоянного канала                                    |
| Недостатки   | Неэффективное использование (паузы)                                          | Переменная задержка                                                          | Большая задержка<br>Буферы на узлах                              |
| Пример       | Телефонная связь (PSTN)                                                      | Интернет (IP)                                                                | E-mail, телеграф                                                 |

## Понятие пакета
Пакет — единица данных, передаваемая в сетях с коммутацией пакетов. Пакет содержит заголовок (header), полезные данные (payload) и иногда хвост (trailer).

### Структура пакета
- Заголовок
- Данные
- Хвост
### Содержимое заголовка
- Адрес источника (Source)
- Адрес назначения (Destination)
- Номер последовательности
- Тип протокола
- Контрольная сумма (Checksum)

### Зачем разбивать на пакеты?
- Эффективное мультиплексирование
- Обнаружение ошибок (на уровне пакета)
- Возможность передачи разными путями
- Снижение задержек (не ждём весь файл)
