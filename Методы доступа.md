#### **1. КЛАССИФИКАЦИЯ МЕТОДОВ ДОСТУПА К СРЕДЕ**
##### **Определение метода доступа**

**Метод доступа к среде передачи данных** — правила и протоколы, определяющие, каким образом несколько станций используют общую физическую среду для передачи информации, чтобы избежать или управлять коллизиями.

### **Две основные категории**
**Детерминированные (управляемые)** - станции получают право передачи по расписанию или эстафете. Гарантированное время доступ

**Недетерминированные (случайные)**- станции пытаются передача без предварительного согласования. Случайный доступ с разрешением коллизий

### **2. НЕДЕТЕРМИНИРОВАННЫЕ МЕТОДЫ**

### **CSMA/CD — Обнаружение коллизий**

**Применение:** Ethernet 10 Мбит/с, Fast Ethernet  
**Стандарт:** IEEE 802.3  
**Принцип:** «Передавай, но слушай»

##### **Алгоритм работы CSMA/CD:**

1. **Слушание среды** — проверка свободы среды
    
2. **Передача кадра** — если среда свободна
    
3. **Контроль во время передачи** — одновременное прослушивание
    
4. **При обнаружении коллизии:**
    
    - Немедленное прерывание передачи
        
    - Отправка jam-сигнала
        
    - Экспоненциальная задержка (binary exponential backoff)
        
5. **Повтор попытки** — максимум 16 попыток

#### **CSMA/CA — Предотвращение коллизий**

**Применение:** Wi-Fi (802.11a/b/g/n/ac/ax)  
**Стандарт:** IEEE 802.11  
**Принцип:** «Слушай, потом передавай»

#### **Алгоритм работы CSMA/CA:**

1. **Слушание среды (sensing)** — в течение DIFS (Distributed Inter-Frame Space)
    
2. **Случайный отсчёт (backoff)** — выбор случайного числа из окна конкуренции (CW)
    
3. **RTS/CTS (опционально):**
    
    - Request to Send (RTS)
        
    - Clear to Send (CTS) — резервирование среды
        
4. **Передача данных**
    
5. **Подтверждение ACK** — от приёмника

#### **3. Сравнение CSMA/CD и CSMA/CA**

| -------------------------------------- | ----------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- |

| **3. #### **4. ДЕТЕРМИНИРОВАННЫЕ МЕТОДЫ (МАРКОВЫЕ/ТОКЕННЫЕ)**
**Марковый (токенный) метод доступа** — детерминированный способ, при котором специальный кадр (маркер/токен) передаётся по кругу. Право передачи принадлежит станции, держащей маркер.

##### **Token Ring (Кольцо с маркером)**

**Стандарт:** IEEE 802.5  
**Топология:** Логическое кольцо

##### **Алгоритм работы Token Ring:**

1. **Циркуляция свободного маркера** — постоянно по кольцу
    
2. **Захват маркера** — станция с данными превращает свободный маркер в занятый
    
3. **Передача данных** — отправка кадра по кольцу
    
4. **Удаление кадра** — отправитель удаляет кадр после полного обхода
    
5. **Выпуск маркера** — освобождение нового свободного маркера
    

**Преимущество:** Гарантированный доступ, отсутствие коллизий  
**Недостаток:** Задержка даже при малой нагрузке

##### **Token Bus (Шина с маркером)**

**Стандарт:** IEEE 802.4  
**Топология:** Физическая — шина, логическая — кольцо

**Принцип:** Маркер передаётся не по физической линии, а адресно между станциями  
**Применение:** Промышленная автоматика (PROFIBUS, CAN-bus), где требуется предсказуемое время доступа (real-time)
